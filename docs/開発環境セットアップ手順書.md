2. 「Sign up」からアカウントを作成
3. メールアドレス・パスワード・ユーザー名を設定して登録

### 1-2. 管理者（久保田）に以下を連絡

```
・GitHubのユーザー名
```

管理者が `yuki1241/Rekka-gyomu-kanri` リポジトリに招待します。

### 1-3. 招待メールを承認

- GitHubから招待メールが届く
- 「Accept invitation」をクリック
- [https://github.com/yuki1241/Rekka-gyomu-kanri](https://github.com/yuki1241/Rekka-gyomu-kanri) にアクセスできれば完了

## STEP 2｜VS Code のインストール

1. [https://code.visualstudio.com](https://code.visualstudio.com) を開く
2. 「Download for Windows（またはMac）」をクリック
3. ダウンロードしたファイルを実行してインストール

## STEP 3｜Claude Code のセットアップ

### 3-1. Anthropicアカウントの作成

1. [https://claude.ai](https://claude.ai) を開く
2. 「Sign up」からアカウントを作成（Googleアカウントでも可）

### 3-2. サブスクリプションの契約

| プラン | 月額 | 推奨 |
|---|---|---|
| Claude Pro | $20/月（約3,000円） | ✅ 通常利用 |
| Claude Max | $100/月（約15,000円） | 大量改修時 |

「Upgrade plan」から **Claude Pro** を選択して契約。

### 3-3. VS Code に Claude Code 拡張機能をインストール

1. VS Code を開く
2. 左側のアイコンから「拡張機能（四角いアイコン）」をクリック
3. 検索欄に `Claude Code` と入力
4. 「Claude Code（Anthropic）」をインストール
5. VS Code を再起動

### 3-4. Claude Code にログイン

1. VS Code の左下または上部メニューから Claude Code を開く
2. 「Sign in」をクリック
3. Anthropicアカウントでログイン

## STEP 4｜リポジトリをローカルに取得

### 4-1. Git のインストール（未インストールの場合）

**Windows の場合：**
1. [https://git-scm.com](https://git-scm.com) からダウンロード
2. インストール（設定はすべてデフォルトでOK）

**Mac の場合：**
- ターミナルで `git --version` を実行 → 自動でインストールが始まる

### 4-2. Node.js のインストール

1. [https://nodejs.org](https://nodejs.org) を開く
2. 「LTS」版をダウンロードしてインストール

### 4-3. リポジトリをクローン

1. VS Code を開く
2. 「ターミナル」→「新しいターミナル」を開く
3. 以下のコマンドを実行：

```bash
# 任意のフォルダに移動（例：デスクトップ）
cd ~/Desktop

# リポジトリをダウンロード
git clone https://github.com/yuki1241/Rekka-gyomu-kanri.git

# フォルダを開く
cd Rekka-gyomu-kanri
```

4. VS Code で「フォルダを開く」→ `Rekka-gyomu-kanri` を選択

## STEP 7｜改修の基本操作

### 7-1. Claude Code の使い方

1. VS Code の左側から **Claude Code アイコン**をクリック
2. チャット欄に改修したい内容を**日本語で入力**する

**入力例：**
```
タスク一覧のページに「優先度」のフィルターを追加してください
```

Claude が自動でコードを修正してくれます。

### 7-2. 変更をGitHubに反映する手順

#### ① 作業用ブランチを作成（必ず最初に実施）

```bash
git checkout -b feature/作業内容の名前
# 例：git checkout -b feature/タスク優先度フィルター
```

#### ② 変更内容を確認

```bash
git status      # 変更されたファイルを確認
git diff        # 変更内容の詳細を確認
```

#### ③ 変更をコミット（保存）

```bash
git add .
git commit -m "タスク一覧に優先度フィルターを追加"
```

#### ④ GitHubにプッシュ

```bash
git push origin feature/タスク優先度フィルター
```

#### ⑤ 管理者（久保田）にレビュー依頼

- GitHubの画面でPull Request（プルリクエスト）を作成
- 管理者が確認してマージ（本番反映）

---

## ⚠️ 作業時のルール

| ルール | 理由 |
|---|---|
| `master` ブランチに直接pushしない | 本番に直接影響するため |
| 作業前に必ず `git pull origin master` を実行 | 最新のコードで作業するため |
| `.env.local` を外部に共有しない | APIキーの漏洩防止 |
| 大きな改修は事前に管理者に相談 | 競合・手戻りを防ぐため |

---

## 🆘 困ったときの対処法

| 状況 | 対処 |
|---|---|
| `npm install` でエラーが出る | Node.jsのバージョンを確認（18以上推奨） |
| ログインできない | `.env.local` の内容を確認 |
| Gitでエラーが出る | `git status` で状態確認後、管理者に連絡 |
| Claude Codeが動かない | VS Codeを再起動、再ログインを試す |

## 📝 用語集

| 用語 | 意味 |
|---|---|
| **リポジトリ** | コードが保存されている場所（GitHub上） |
| **クローン** | GitHubからコードをPCにダウンロードすること |
| **ブランチ** | 本番に影響せず作業できる「作業スペース」 |
| **コミット** | 変更内容を記録・保存すること |
| **プッシュ** | ローカルの変更をGitHubにアップロードすること |
| **プルリクエスト** | 変更内容をレビュー・本番反映依頼すること |
| **マージ** | ブランチの変更を本番（master）に取り込むこと |
| **環境変数** | APIキーなどの機密情報を管理するファイル |
